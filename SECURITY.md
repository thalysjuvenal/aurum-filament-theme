# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, report it privately using [GitHub Security Advisories](https://github.com/thalysjuvenal/aurum-filament-theme/security/advisories/new) — click **"Report a vulnerability"** on the repository's Security tab. This is the preferred and only supported reporting channel.

Include as much detail as possible:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof-of-concept if available.
- The affected version(s) of the package.

## Response Expectations

This project is maintained independently, on a best-effort basis. There is no guaranteed SLA, but reported vulnerabilities will be triaged and addressed as promptly as possible. You'll receive an acknowledgement through the Security Advisory thread, and a fix or mitigation plan once the issue has been assessed.
